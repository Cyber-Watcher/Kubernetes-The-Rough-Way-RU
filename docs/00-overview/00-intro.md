## **0.1. Цель проекта**

Цель проекта **Kubernetes The Rough Way** — создать полностью self‑hosted, production‑ready Kubernetes‑кластер **вручную**, без kubeadm, без автоматизации, с полным пониманием каждого компонента.

Этот проект:

- раскрывает внутреннее устройство Kubernetes  
- показывает, как работает control plane  
- объясняет, как строится HA‑кластер  
- учит работать с PKI, TLS, etcd, CNI, kubelet, kube-proxy  
- формирует инженерное мышление уровня Senior DevOps/SRE  

Мы не просто «поднимаем кластер» — мы **строим его кирпич за кирпичом**, документируя каждый шаг.

---

## **0.2. Что такое Kubernetes The Rough Way**

Это:

- **не** копирование команд из Kubernetes The Hard Way  
- **не** быстрый способ получить кластер  
- **не** учебник для новичков  

Это:

- инженерное руководство  
- глубокое изучение архитектуры  
- практическая реализация HA‑кластера  
- документация уровня production  
- понимание, как работает Kubernetes «под капотом»  

Мы создаём кластер, который:

- можно расширять  
- можно восстанавливать  
- можно поддерживать  
- можно использовать как основу для автоматизации  

---

## **0.3. Чем этот проект отличается от Kubernetes The Hard Way**

| Kubernetes The Hard Way | Kubernetes The Rough Way |
|-------------------------|---------------------------|
| копирование команд | понимание архитектуры |
| минимум объяснений | глубокие инженерные объяснения |
| нет HA | полноценный HA control plane |
| нет PKI‑архитектуры | собственная PKI с Root CA, etcd CA, Kubernetes CA |
| нет LVM | продакшн‑разметка дисков |
| нет ingress, DNS, storage | полный production stack |
| нет документации | полноценное руководство |

Мы не повторяем KTHW — мы создаём **реальный production‑кластер**, как это делает Senior DevOps.

---

## **0.4. Требования к читателю**

Проект рассчитан на инженера, который:

- уверенно работает в Linux  
- понимает сети (IP, CIDR, маршрутизация)  
- знает основы контейнеризации  
- знаком с YAML и kubectl  
- хочет понять Kubernetes глубоко  

Если ты читаешь этот документ — ты уже подходишь.

---

## **0.5. Что будет построено в итоге**

Мы создадим:

- **3‑нодовый HA control plane** (stacked etcd)  
- **2 worker‑ноды**  
- **VIP с keepalived + haproxy**  
- **Calico CNI**  
- **containerd runtime**  
- **CoreDNS**  
- **ingress‑контроллер**  
- **metrics‑server**  
- **storage provisioner**  
- **полную PKI‑инфраструктуру**  
- **полную документацию**  

И всё это — вручную, без kubeadm.
