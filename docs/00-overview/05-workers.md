# **5. Worker Nodes — детальный обзор**

Этот документ описывает **все компоненты worker‑нод**, их роли, взаимодействие с control plane, сетевую модель, жизненный цикл Pod’ов и то, как worker‑ноды вписываются в архитектуру Kubernetes The Rough Way.

Worker‑ноды — это **рабочая плоскость (Workload Plane)**.  
Именно здесь запускаются Pod’ы, контейнеры, сетевые агенты и сервисы приложений.

---

# **5.1. Роль worker‑нод в Kubernetes**

Worker‑ноды выполняют:

- запуск Pod’ов  
- управление контейнерами  
- сетевую маршрутизацию  
- реализацию Service networking  
- взаимодействие с control plane  
- применение NetworkPolicy  
- выполнение workload’ов (приложений)  

---

# **5.2. Компоненты worker‑ноды**

Каждая worker‑нода содержит четыре ключевых компонента:

1. **kubelet**  
2. **containerd**  
3. **kube-proxy**  
4. **CNI (Calico)**  

Эти компоненты обеспечивают полный цикл работы Pod’ов.

---

# **5.3. kubelet — агент ноды**

kubelet — это главный агент worker‑ноды.

### Функции:

- получает PodSpec от kube-apiserver  
- создаёт Pod sandbox  
- запускает контейнеры через CRI  
- следит за состоянием Pod’ов  
- обновляет статус Pod’ов в API  
- управляет readiness/liveness probes  
- монтирует тома  
- взаимодействует с CNI  

### Особенности в нашем кластере:

- работает на всех worker‑нодах  
- подключается к API через VIP  
- использует TLS‑сертификаты  
- взаимодействует с containerd через `/run/containerd/containerd.sock`  

---

# **5.4. containerd — контейнерный runtime**

containerd — это runtime, который запускает контейнеры.

### Функции:

- создаёт контейнеры  
- управляет их жизненным циклом  
- предоставляет CRI интерфейс kubelet’у  
- использует runc для запуска контейнеров  

### Особенности в нашем кластере:

- используется как единственный runtime  
- работает как systemd‑сервис  
- взаимодействует с kubelet через gRPC  

---

# **5.5. kube-proxy — реализация Service networking**

kube-proxy — это сетевой компонент, который реализует Kubernetes Service.

### Функции:

- создаёт iptables/ipvs правила  
- перенаправляет трафик к Pod’ам  
- обеспечивает ClusterIP, NodePort, LoadBalancer  
- синхронизирует Endpoints  

### Особенности в нашем кластере:

- работает в режиме **ipvs**  
- получает конфигурацию от API  
- обновляет правила при изменении Endpoints  

---

# **5.6. Calico (CNI) — Pod‑сеть и BGP‑маршрутизация**

Calico — это CNI‑плагин, который обеспечивает:

- Pod‑сеть  
- маршрутизацию между нодами  
- NetworkPolicy  
- IPAM  

## Мы используем **Calico BGP mode (без overlay)**

Это значит:

- каждый узел — BGP speaker  
- каждый узел анонсирует свой PodCIDR  
- маршруты Pod → Pod — чистый L3  
- нет VXLAN  
- нет encapsulation overhead  

### Особенности в нашем кластере:

- Pod CIDR: `10.244.0.0/16`  
- worker‑01: `10.244.1.0/24`  
- worker‑02: `10.244.2.0/24`  
- маршруты анонсятся через BGP  

---

# **5.7. Жизненный цикл Pod’а на worker‑ноде**

Жизненный цикл Pod’а выглядит так:

```
1. Scheduler выбирает ноду
2. kubelet получает PodSpec
3. kubelet создаёт Pod sandbox
4. kubelet вызывает CNI ADD → создаётся сеть Pod’а
5. kubelet вызывает containerd → запускаются контейнеры
6. kubelet обновляет статус Pod’а в API
7. kube-proxy обновляет правила Service
8. Calico обновляет маршруты
```
---

# **5.8. Сетевые взаимодействия worker‑ноды**

Worker‑нода взаимодействует с:

### **kube-apiserver**
- получение PodSpec  
- обновление статуса  
- получение конфигурации kube-proxy  
- получение NetworkPolicy  

### **etcd (косвенно через API)**
- worker‑нода не общается с etcd напрямую  

### **другими worker‑нодами**
- через BGP маршруты Calico  

### **control plane**
- kubelet → apiserver  
- kube-proxy → apiserver  
- Calico → apiserver  

---

# **5.9. Почему worker‑ноды не являются частью control plane**

Worker‑ноды:

- не принимают решений  
- не хранят состояние  
- не управляют кластером  
- не выполняют контроллеры  

Они только:

- запускают workload  
- обеспечивают сеть  
- обеспечивают контейнеры  

---

# **5.10. Итоговая схема worker‑ноды**


```text

┌──────────────────────────────────────────────┐
│          kube-apiserver (VIP / CP)           │
└──────────────────────────────────────────────┘
                  ▲           ▲
                  │           │
          kubelet │           │ kube-proxy
                  │           │
                  │           │
┌─────────────────┴───────────┴──────────────┐
│                 Worker Node                │
│                                            │
│  ┌──────────────┐                          │
│  │   kubelet    │──────────────────────────┘
│  └──────────────┘   (talks to apiserver)   │
│          │                                 │
│          ▼                                 │
│  ┌──────────────┐                          │
│  │  containerd  │                          │
│  └──────────────┘                          │
│                                            │
│  ┌──────────────┐                          │
│  │  kube-proxy  │──────────────────────────┘
│  └──────────────┘   (watches Services/EP)  │
│                                            │
│  ┌──────────────┐                          │
│  │   Calico     │ ←→ BGP → другие ноды     │
│  └──────────────┘                          │
└────────────────────────────────────────────┘
```
